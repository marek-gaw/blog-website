Standard setup for presentations

Posted on śro 13 maja 2020 in General • 1 min read

Original idea was created by Michał Gałka, and is available on Github.

My main goal is to create my presentations in Markdown and generate output in a form, which is the most suitable: it might be html or pdf. With this approach, I can focus on the content, without thinking about how exactly it will be presented on a page. This is also a way to overcome a desire to squash too much on a single slide.

Content is created in single Markdown document and output is generated with Pandoc.

In case of more interactive presentations, I use great HTML/JS framework: reveal.js. In my setup Pandoc uses markdown document as an input and generates complete website.

As an example, you can take a look at presentation given at CEHUG last year: ROS: o robotach słów kilka
