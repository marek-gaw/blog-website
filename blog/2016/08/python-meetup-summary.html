Demo 2¶

Second demo was a little bit more complicated. I wanted to show how ROS may “talk” with non-ROS applications.

ROS provides at least two solutions: [ros_serial] and [rosbridge].

First is dedicated to communication with devices like Arduino, over UART port, second is better for applications, which are able to communicate over WebSockets.

I shown how to receive data from ROS topic with Python client (using websocket-client package) and with webpage with JS library.

Files:

Those files were taken from rosbridge tutorial, and this thread at Stack Overflow.

websocket_page, websocket_python
